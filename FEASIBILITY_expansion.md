# 可行性研究：自選股納入訓練池 與 Model v2 時程

> 2026-07-17｜結論先講，僅列有憑據項

## 結論

**可行，但有條件**：自選股（如 6669 緯穎、2337 旺宏）納入訓練池不是
「加了就變準」，而是要過三道閘。第一道（擴充池特徵稽核）**零風險、
現在就能跑**，一條指令：

```
python feature_signal_audit.py --holdings holdings.csv --v2 --extra-symbols 6669,2337
```

## Model v2 什麼時候建？——證據導向，非日曆導向

**Model v1 前瞻裁決時點（可估）**：獨立樣本現況 10/30；逐檔間隔≥5 交易日
計數，十檔每 5 個交易日約 +10 → **約 2026-07-30 前後達 30**，屆時對
Model v1 做一次性裁決（預期：確認無判別力 → 歸檔，與 holdout AUC≈0.494
一致）。附註：同日十檔的預測橫斷面相關，裁決時將以保守方式處理
（叢集層級檢定），不以名目 30 膨脹信心。

**Model v2 啟動條件（三道閘，過一道才開下一道）**：
1. **擴充池特徵稽核**（方案A，上面那條指令）：在「前十大+自選」的更大
   開發集上，任何特徵過 Holm + VG-7 真偽篩 → 才有建模的理由。
   全軍覆沒 → 擴充池不改變「無訊號」結論，Model v2 無啟動依據。
2. **折內回歸+IC 評估**：閘1 存活特徵在 walk-forward 折內以連續報酬
   檢驗（非二元分類，避免離散度假象）。
3. **前瞻 30 獨立樣本**：新模型上線後由每日自動化累積，達標一次性裁決。

**一句話回答「何時建模」**：閘 1 現在就能跑；若有存活特徵，閘 2 約需
一週開發+驗證；Model v2 上線後再等前瞻累積（以擴充池 12 檔計，約
3–4 週達 30）。若閘 1 全滅，誠實結論是「擴充池亦無訊號，不建 v2」。

## 風險清單（憑據）

| 風險 | 憑據 | 對策 |
|---|---|---|
| **選股偏誤（最大）** | 6669/2337 是「表現好才被注意」的股票——用它們擴充=把倖存者偏誤注入訓練 | 顯著發現須以**客觀納入規則**（如「2024-01-01 市值前 N 大且日均量>門檻」的全名單）重跑確認才可宣告；稽核輸出已內建此警語 |
| 同質性 | 6669/2337 與現池同屬半導體/AI 供應鏈，樣本相關性高、邊際資訊有限 | 預期管理：更可能是「更大樣本更確定無訊號」而非變出訊號 |
| 資料長度 | 6669 上市於 2019（1,777 根 vs 池內約 2,451 根） | 獨立樣本貢獻較少；稽核自動處理，無需特別動作 |
| holdout 已多次評估 | N=15 假陽性教訓 | 擴充池**不碰舊 holdout**；最終裁決一律走前瞻 |
| 治理 | 訓練池變更不可被排程觸發（已制度化） | 走正式流程：holdings 更新 + 全套 VG + 前瞻 |

## 不建議的做法（明確排除）

「加股票 → 重訓 → 看 holdout AUC → 不滿意再加」——這是在 holdout 上做
多重比較，幾乎保證製造假訊號（L 系列教訓的同族錯誤）。
