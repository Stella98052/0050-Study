# 第三階段使用指南 — 即時面板與前瞻驗證（v3.0）

> 本系統僅供研究與教育用途，歷史回測績效不代表未來表現，不構成投資建議。

## 一、這一階段交付什麼

在 Phase 2 完成三線科學結論（規則訊號無優勢／成本攤薄不成立／模型無判別力）
之後，第三階段把系統做成可日常操作的面板，並建立 Model v2 的前瞻驗證管線。
**面板不依賴訊號有效性**：VG 狀態卡如實顯示每一關（含 VG-3❌、VG-6❌），
模型無判別力時預測卡強制掛紅色警語，只作架構演示。

三支新程式：
- `app_streamlit.py` — Web 面板（下拉切股／K線+波浪雙版轉折／MV潮汐副圖／
  預測卡／VG 狀態小卡／前瞻進度）
- `daily_update.py` — 每日收盤後更新（沿用 phase1 抓取+VG-1，輸出預測、
  累積前瞻紀錄）
- `src/dashboard/` — 六個純函式模組（資料/圖表/模型/VG卡/前瞻紀錄），
  全部委派 phase1/phase2 既有函式，單一事實來源，皆有鎖定測試

## 二、安裝與啟動

```
pip install -r requirements.txt        # 已含 streamlit 1.59.2 / plotly 6.9.0
streamlit run app_streamlit.py
```
瀏覽器自動開啟 http://localhost:8501 。左側下拉選 0050 前十大持股（顯示「代號 名稱」，如 2330 台積電；名稱取自 holdings.csv 的 name 欄，與持股清單同源同基準日）。
首次載入某檔會抓官方日K（禮貌延遲，較慢屬正常）；同日重載走 1 小時快取。

## 三、每日更新與前瞻驗證（Model v2 最終閘）

```
python daily_update.py --holdings holdings.csv
```
每交易日收盤後執行一次：更新資料→VG-1 檢查→輸出各股 P(未來5日淨報酬>0)
→append 至 `data/predictions.csv`（同股同末根K去重，紀錄不可竄改）。

**前瞻裁決規則（預先宣告，不得事後更改）**：predictions.csv 的「獨立樣本」
（逐檔間隔≥5日，select_independent_dates）累積 ≥30 後，才對 Model v2 做
一次性統計裁決。達標前面板進度條顯示累積狀況，任何人不得偷看下結論。
以現有模型（AUC≈0.494）預期前瞻仍不顯著；此管線的價值是「用真正未見過
的未來資料做最終確認」，而非沿用已被多次評估的 holdout。

Windows 工作排程器（規格④只需說明，範例）：
```
schtasks /Create /SC WEEKLY /D MON,TUE,WED,THU,FRI /TN "0050DailyUpdate" ^
  /TR "cmd /c cd /d C:\path\to\phase1 && python daily_update.py --holdings holdings.csv" /ST 15:30
```
（TWSE 盤後資料約 15:00 齊備；過早執行會取到前一交易日，見資料新鮮度教訓）

## 四、面板五要件對規格

| 規格要求 | 面板落實 |
|---|---|
| 下拉切換前十大個股 | 側欄 selectbox（讀 holdings.csv）|
| 主K線+波浪轉折(1..5,A,B,C)雙色 | Plotly OHLC；retro灰(禁入模型)/realtime藍(可決策) |
| 副圖 MV 潮汐柱+潮汐線 | 成交量柱 + 5/13/20MV；13MV核心否決線粗線 |
| 預測卡(看多空/信度/波浪警示/風險聲明) | 四欄卡+13MV否決警示+固定聲明 |
| VG 驗證狀態小卡 | VG-1~VG-6 六卡，未通過顯示❌不省略 |
| 風險聲明固定顯示 | 頂部 st.warning 常駐 |

## 四之二、面板兩類資訊的區分（重要，避免誤讀）

面板有兩種本質不同的資訊，已在版面上明確分區：

**本股即時值（隨股票變動）** — 預測卡的方向、P(未來5日淨報酬>0)、
realtime 波浪位置、13MV 核心否決線。切換股票時這些會變。

**全模型層級（對每檔相同）** — VG-1~VG-6 驗證關卡。這六關檢驗的是
「整個模型與規則訊號的統計可信度」；十檔以同一個模型訓練，所以無論
選哪一檔，六關狀態都一樣。**每檔顯示相同是設計，不是 bug** ——它回答
的是「這組模型可不可信」，不是「這一檔今天怎樣」。若它隨股票變動反而錯了。

VG-3❌／VG-6❌ 是 Phase 2 的誠實科學結論（規則訊號無優勢、模型無判別力），
面板如實顯示，代表驗證系統正確攔下了假訊號，非故障。

## 四之三、自選股票（v3.3）

側欄「自選股票」欄輸入任意台股代號（0050 以外亦可，例如持倉 2337 旺宏）
按 Enter 加入；「移除自選」可刪。加入後出現在「選擇股票」下拉，標「（自選）」。

**自選股的能與不能（治理邊界）**：
- ✅ 有效：K線、波浪轉折（retro/realtime）、MV 潮汐——全部走 TWSE 官方
  日K與 phase1 既有公式計算，與前十大同一套引擎，可據方法論人工研判
- ❌ 不適用：模型預測卡。模型僅以 0050 前十大訓練，未見過自選股，
  對它預測無統計意義，故自選股時預測卡隱藏並顯示說明
- VG 驗證關卡是「全模型層級」，與自選股無關（模型本身不涵蓋自選股）

自選清單僅存於本次瀏覽器工作階段（關閉即清空），不寫入 holdings.csv，
不影響 run_phase2 的訓練池——訓練池的變更需正式更新 holdings 並重跑。

## 五、誠實性設計（治理準則延伸至面板）
- 無 phase2 報告 → VG 卡全標「⚠尚未執行」，**絕不默默顯示全綠**（有測試鎖定）
- VG-6 未通過或狀態未知 → 預測卡掛紅色蓋過警語，數字僅供演示
- 模型特徵欄與當前不符 → 明確報錯，不靜默套用（serialize 層 assert）
- 13MV 下彎 → 面板獨立標示絕對否決，凌駕模型/波浪判讀（方法論鐵律）
