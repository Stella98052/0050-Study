## L1：TWSE 官方 API 會回錯月份的資料（最重要）
- **現象**：VG-1 報 2454 重複日期 21 筆、2308 重複 20 筆（≈恰一個月交易日數）
- **診斷**（diagnose_duplicates.py）：2454 的 201905.csv 內容實為 202411；
  2308 的 201709.csv 內容實為 202005。抓取為單執行緒逐月、逐股獨立資料夾，
  排除本地錯位 → **是 TWSE STOCK_DAY 對某月請求回傳了另一個月的內容**
- **防護（三層）**：①越月過濾：回應/快取列不屬於請求月→過濾+警示
  ②合併去重：仍重複→警示+保留首筆 ③診斷工具點名污染檔供刪除重抓
- **原則**：官方 API 也會回錯資料；快取層必須驗證「回應內容屬於請求範圍」，
  異常要可見（不靜默），並提供自我診斷工具

## L2：當月快取凍結（每日更新的隱形殺手）
- **現象**：202607.csv 於 7/11 首抓後被永久快取，整個七月沿用，

- **原則**：「進行中」的資料不可永久快取；快取策略須區分已定案/未定案期間

## L3：官方休市日表僅涵蓋當年度

- **修法**：校驗僅於休市日表實際涵蓋期間 [min,max] 內進行，範圍外明示「不校驗」
- **原則**：使用參照資料前先確認其涵蓋範圍；範圍外標註未知，不可誤報

## L9：回測權益曲線的三層失真（v2.0→v2.3 演進，2026/7/12 單日修三次）
- **v2.0**：逐筆連乘 → 重疊持有被虛構成序列複利（holdout 1,677萬%）
- **v2.1**：等權日籃連乘 → 量級仍被時間壓縮 N 倍（+1948%）
- **v2.2**：日等效 (1+r)^(1/N) 鏈乘 → 指數律恆等於 (∏(1+r))^(1/N)，
  修了量級沒修結構——它不對應任何可執行交易路徑
- **v2.3（定案）**：不重疊交易（間隔≥N，同 VG-4 規則）原始報酬直接連乘
  = 真實可執行的單部位策略路徑，無需換算
- **原則**：重疊觀測不能序列鏈乘；權益曲線的每個數字必須對應一條
  「實際可能發生」的交易路徑，否則就是尺度修正掩蓋結構錯誤

## L10：退化案例當鎖定測試 = 自我印證陷阱（外部審查抓到，雙方都踩過）
- v2.2 的鎖定測試用「連續五日報酬全部相同」——恰是錯誤方法會蒙對的退化案例
- 外部審查者第一版驗證例（對稱交替±報酬）也巧合蒙對，自承踩同類坑
- **正解**：鎖定測試必須用「不對稱、不成組」的案例
  （[+10%,0,0,0,0,+5%]：正確 15.5% vs v2.2 錯誤 2.92%，差異立現）
- **原則**：設計鎖定測試時先問「什麼輸入會讓錯誤方法也算對？」並刻意避開

## L11：外部審查的兩種正確結局（與 c1 乘數案對照）

## L13：VG-3 偽重複——統計檢定的樣本獨立性（2026/7/13 外部審查抓到，驗算採納）
- **現象**：N 網格研究中 N=10/15/20/30 四點齊釘 p=0.0，模式本身即可疑線索
  （真實訊號漸強應有連續變化，不會整批釘 0）
- **根因**：VG-3 餵入未篩選的交易層序列（N=10 時數百筆重疊觀測），
  重疊樣本在趨勢中群聚正相關 = 偽重複（pseudo-replication），
  人為收窄虛無分布 → 壓低 p 值。受控驗算：同資訊灌 8 倍重複，p 0.266→0.054
- **更嚴重的定性**：定案4 原文早已寫明「統計獨立層輸出的樣本數才是
  VG-3 與 VG-4 唯一採用的計數依據」——實作只讓 VG-4 遵守、VG-3 漏了，
  屬違反已定案規則，非單純技術瑕疵
- **修法**：新增 independent_return_series 共用核心（同日等權合併→
  ≥N 篩選），VG-3 一律先篩再檢定；權益曲線同源委派（L12 落實）
- **連帶規則**：VG-4 未達門檻時 VG-3 結果標「未決(VG-4蓋過)」，
  與 holdout Sharpe 同一待遇——治理規則必須覆蓋每一條新流程，
  新腳本不能成為規則的法外之地
- **原則**：權益曲線修過的偽重複，同樣存在於統計檢定的獨立性假設；
  同一類錯誤會在不同位置重複出現，修一處時要問「還有哪裡用了同樣的假設」

## L14：多頭樣本中的長持有幻覺——隨機進場基準
- 淨報酬隨 N 由負轉正不能證明訊號優勢：任何長多策略在多頭樣本中
  拉長持有都會吃到市場飄移（holdout Alpha −33.66% 已是同一課）
- **修法**：N 網格加入 baseline（同池同 N 任一日進場均值）與
  edge = 訊號獨立均 − baseline；能支持假說的是 edge 隨 N 擴大
- **原則**：報酬數字必須相對於「不含訊號的同構基準」呈現，
  絕對值由市場環境決定，邊際才是訊號的貢獻

## L15：趨勢結論的預先宣告資格（v2.5，2026/7/13）
- N 網格回答的是「edge 隨 N 如何變化」——比單點更弱、更易被雜訊污染的問題
- **預先宣告規則（寫在擴大樣本之前，防「規則寫在結果之後」）**：
  ①edge 一律配對計算（逐筆扣同進場日全池均值）並附 bootstrap 95% CI，
    相鄰 N 的 CI 大幅重疊 = 雜訊連線的客觀判準
  ②至少 3 個「相鄰」N 同過 VG-4 門檻才有趨勢結論資格（程式化判定）
  ③樣本分母擴充優先序：0050 全體成分股（~50檔，不動歷史區間）→
    仍不足才延長年限（成本高且加重早期成分股存活偏誤）

## L16：同屏矛盾是最好的偵錯訊號——VG-3/VG-4 同源化（v2.6，2026/7/13）
- **現象**：同一次執行輸出「VG-3 檢定樣本 217 筆」與「VG-4 獨立訊號數 35」
  ——依定案4 兩者本該同源同數，同屏差 6 倍即定義漂移的鐵證（L12 預言成真）
- **根因①（bug）**：count_statistically_independent_signals 假設輸入按時間
  排序，但 run_phase2 餵入「依股票分塊 concat」序列——跨股邊界日期倒退，
  greedy 間隔算出負數，後續股票事件幾乎全被跳過 → VG-4≈只算第一檔
  （樣本外「獨立3」同為低估）
- **根因②（定義分歧）**：VG-3 用日期直接 greedy、VG-4 用事件合併後 greedy
  （定案4 原文為後者），兩套各自演化
- **修法**：canonical_independent_samples 正典管線（逐日聚合 any →事件→
  ≥N→代表日報酬均值），VG-3 與 VG-4 一律取自此函式，同源同數以測試鎖定；
  多股未排序輸入的回歸測試釘入
- **定義分工（刻意保留兩層，各自單一來源）**：統計層=事件法（一段連續
  訊號episode=1樣本，定案4）；路徑層=可執行再進場（日期 greedy，權益曲線
  用）——連續12日episode、N=5：統計層1樣本 vs 路徑層3筆，測試鎖定防互改
- **原則**：兩個本該相等的數字同屏出現不等，永遠優先當 bug 查，
  而不是各自解釋；輸出設計應刻意讓同源數字並列以自我暴露矛盾

## L17：N 網格的兩道統計閘與範疇標示（v2.7，2026/7/13 審查三採納一駁回）
- **B1 多重比較（採納）**：5 個 N 各自檢定，純雜訊下至少一次假顯著機率
  22.6%——單點 raw p 顯著不算數。Holm 校正入庫（免新依賴手算逐步法）；
  實跑驗算：N=15 raw 0.008 → adj 0.040 存活，其餘全不顯著
- **B2 趨勢檢定（採納）**：zigzag 形狀不在原兩分支判讀準則內 → 禁止
  肉眼連線，edge 對 N 線性迴歸「斜率>0 且 p<0.05」才可宣稱趨勢；
  實跑：斜率+0.0915%/日、p=0.0596、R²=0.745 → 未達顯著
- **A1 標籤寫反（採納）**：181 是「統計獨立層計數」（事件法篩選後），
  不是「事件法計數」（事件數是210）——輸出字串抽成可測函式並鎖定
- **A2 相等斷言（駁回，範疇標示採納）**：路徑獨立51=模型策略（LightGBM
  預測）路徑層；VG-4 獨立21=規則訊號統計層——不同策略×不同層，本就
  不該相等，強制相等斷言會錯誤炸開。真問題是輸出未標明範疇 → 修為
  明確標示【模型策略】/【規則訊號】，並為模型策略補上自身的統計層
  計數與蓋過閘
- **原則**：審查建議逐項驗算後可「部分採納」——診斷對、藥方錯的情況
  存在；範疇混淆的正解是標示與分閘，不是強迫不同對象相等

## L18：獨立性定義必須綁定對象——事件法對密集策略的語意退化（v2.8）
- **現象**：holdout 模型策略「路徑獨立51 vs 統計獨立1」，50 倍落差
- **根因**：非 bug、非兩套 N——模型策略每日重選、跨10檔，逐日 any() 聚合
  後 100% 為 True → 事件法把整段 holdout 併成 1 個事件。事件法（定案4）
  的設計對象是「稀疏 episode 型規則訊號」，套到密集策略上語意退化
- **修法（定義綁定對象）**：
  稀疏規則訊號 → 事件法（VG-3/VG-4，定案4 原意）
  密集模型策略 → 非重疊窗口法（統計樣本=權益曲線路徑層，同一呼叫同源，
  runtime 斷言鎖定），並對其跑真正的 bootstrap 檢定，非只靠計數閘
- **診斷工具**：independence_divergence_report 並排兩法計數+訊號日占比，
  自動判定適用方法（占比≥50%→path）
- **VG-2 交叉驗證（審查詢問）**：程式追溯確認主/對照兩側皆為
  規則訊號 × 同一 compute_backtest_metrics —— 對稱、未被模型策略
  分歧污染；45.71% 之比較基礎已標明於輸出
- **原則**：「同源」的正確單位是「同一對象」——不同對象各自同源、
  且輸出必須說明為何合理地不同；被蓋過≠結案，分歧要解釋到底

## L19：聚合層級誤判與 VG-6 的誕生（v2.9，2026/7/14）
- **審查推論的修正**：「訊號日占比100% ⇒ 模型退化」混淆了聚合層級——
  逐列選中率 41.5%（997/2400，明顯有篩選）的模型，跨十檔聚合後
  「任一檔/日」占比 = 1−(1−0.415)^10 ≈ 99.5%，100% 是多股池數學必然，
  不能單獨作為退化證據
- **但問題本身正確且採納**：機率分布形狀與判別力此前五道關卡皆未覆蓋
  → 新增 VG-6（逐列機率分布 std/中間帶/單邊率 + 免 sklearn 的
  Mann–Whitney AUC），退化與否用資料回答、不用推論猜；
  輸出附「聚合占比為數學必然」的防誤判註記
- **措辭分級（採納）**：趨勢迴歸 p=0.0596 屬「臨界未顯著」，與六閾值
  全平坦那種鐵證級否定必須語氣區隔——結論的份量要跟證據強度成正比
- **原則**：審查的「診斷推論」與「建議的檢查」可以分開評價——推論
  錯了不代表檢查不該做；健檢項目寧可多建，判定交給資料

## L26：面板預測全股失敗——對齊檢查比對錯層級（v3.1 修，2026/7/15）
- **現象**：面板預測卡每一檔股票都報「特徵欄不一致，缺少 wave_1…wave_C,
  wave_unknown；拒絕套用模型」——全股一致失敗即「與個股無關的結構問題」
- **根因**：模型 wrapper.predict_proba 設計為「吃原始特徵 df、內部 _encode
  自動 one-hot 波浪標籤」（固定類別集 _WAVE_CATS）。但 model_service 的
  assert_feature_alignment 拿模型記錄的『展開後』特徵名（wave_1…）去比對
  面板送的『原始』欄（wave_label_realtime），在 wrapper 編碼之前就檢查 →
  必然報缺欄
- **修法**：對齊檢查改驗證原始 FEATURE_COLS 齊備（wrapper 會自行展開），
  非展開後欄名；真缺原始欄仍明確 raise（不靜默）。端到端鎖定測試：
  原始特徵矩陣→出機率 + 缺原始欄→報錯
- **原則**：跨階段介面（訓練序列化 vs 推論載入）的「特徵契約」必須明確
  定義比對層級（原始欄 or 編碼後欄）；wrapper 內建編碼時，對齊檢查要在
  編碼前的層級進行。全股一致的失敗永遠指向結構/契約問題，非資料問題
